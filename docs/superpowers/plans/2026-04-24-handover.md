## Build / deploy workflow (gotchas)

- Build: `./build-with-vs.bat` (loads `vcvars64.bat` then runs `build.bat`). Don't use `build.bat` directly from the shell in this session ‚Äî `cmake` isn't on the default PATH.
- Deploy: `./deploy.ps1` (run from PowerShell, **not** bash ‚Äî bash tries to interpret PowerShell syntax). Auto-versions as `AI Synth vN.vst3` in `C:\Program Files\Common Files\VST3\` and renames the inner DLL to match the bundle (VST3 spec requirement). Side-by-side versions let the user A/B in the host.
- **Reaper file lock**: Reaper holds the VST3 DLL while loaded, causing `LNK1104: cannot open 'AI Synth.vst3'` on rebuild. Before building, check: `tasklist //FI "IMAGENAME eq reaper.exe"`. If running, ask the user to close it ‚Äî don't kill it.
- The `/deploy-vst` slash command wraps `deploy.ps1`.

## Design conventions already established

- Use `Tokens::panel` for the main surface, `panel2` for interior lifted panels, `panel3` only where needed. Never raw hex literals outside of black/dark intermediates in gradients.
- Radii: `radiusMd` (6 px) for plot frames and small affordances, `radiusLg` (10 px) for panels.
- Active state = `accent` (violet `#C693E8`). Hover = `accent.withAlpha(0.10‚Äì0.25f)`.
- Label typography: `Fonts::sans(Font::body)` for controls; `Fonts::mono(Font::tiny)` for readouts/hints.
- Plot frames: always go through `drawPlotFrame()` in `PlotFrame.h` ‚Äî don't re-roll the gradient.
